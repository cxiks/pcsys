    Внедрение и поддержка компьютерных систем

1. Определение компьютерных систем
Компьютерные системы — это комплекс:

компьютерного и активного сетевого оборудования,

системного и офисного программного обеспечения,
необходимый для функционирования автоматизированной системы предприятия.

2. ГОСТ Р ИСО/МЭК 12207-2010
Стандарт по системной и программной инженерии.

Устанавливает процессы жизненного цикла программных средств.

Определяет процессы для:

приобретения,

поставки,

разработки,

применения,

сопровождения,

прекращения использования ПО.

3. Жизненный цикл ПО
Период от разработки до эксплуатации программного обеспечения.

4. Подходы к внедрению компьютерных систем
Контракт с крупной компанией
✅ Опыт, наработки
❌ Высокая стоимость, возможная текучка кадров, качество специалистов может не соответствовать имени.

Приглашение региональной IT-компании
✅ Проект будет приоритетным
❌ Риск смены руководства/специалистов на долгих проектах.

Внедрение силами собственного IT-отдела
✅ Экономия, постоянная связь, личный контроль
❌ Зависимость от решений пользователей и руководства, возможны ошибки.

Приглашение эксперта
✅ Экономия, узкий специалист
❌ Зависимость от одного человека, необходимость высокой организованности коллектива.

5. Этапы внедрения компьютерных систем
Обследование
Диагностика процессов предприятия.
Длительность: 1 неделя – 1 месяц (в среднем 2 недели).

Разработка технического задания (ТЗ)
Описание справочников, алгоритмов, отчётов, АРМ, прав доступа.
Длительность: 1–3 месяца (в среднем 1,5–2 месяца).

Настройка системы
Формирование функционала, настройка рабочих мест, ввод пользователей, настройка прав.
Длительность: 1–1,5 месяца.

Тестирование
Подготовка демо-примера, проведение тестов, исправление ошибок.
Длительность: около 2 недель.

Опытная эксплуатация
Работа с реальными данными параллельно со старой системой.
Длительность: около 1 месяца.

Промышленная эксплуатация
Полный переход на новую систему, организация техподдержки.
Длительность: около 1 месяца.

6. Основные проблемы при внедрении
Недостаток прозрачности разработки.

Недостаток контроля (срывы сроков, превышение бюджета).

Недостаток мониторинга процесса разработки.

Недостаточная надёжность системы разработки (сложность поиска и исправления ошибок).

Отсутствие гарантий на программы.
